# 画面仕様: 設定 (Settings)

アプリ全体のデータ管理と、ヘルプ、開発者向けのオプションを提供する画面。

## 1. データ管理 (Data Management)

### 一括インポート (Batch Import)
- **機能**: JSONファイルからデータを一括で取り込む。
- **データ形式**: JSON形式
- **操作**: 
  1. 「一括インポート」をタップすると、データの種類（スケジュールデータ / プロフィールデータ）を選択するダイアログが表示される。
  2. 種類を選択すると、ファイルピッカーが開く。
  3. JSONファイルを選択すると、インポートが実行される。
  4. 完了後、インポートした件数がトーストで表示される。
- **バリデーション**:
  - プロフィールデータ: 特になし（名前の重複も許可され、新規追加される）。
  - スケジュールデータ:
    - 形式（SINGLES/TEAMS）は必須。
    - トリオ（TEAMS）形式の場合、チーム結果が必須。
    - シングルス（SINGLES）形式の場合、チーム結果はnull（空）である必要がある。
- **自分設定の挙動**:
  - インポートデータ内に「自分（isSelf）」が1人以上含まれている場合、現在のアプリ内の「自分」設定は一旦リセットされる。
  - インポートデータ内の設定に従い、最後に出現した「自分（isSelf: true）」のファイターが最終的な「自分」として設定される。
  - インポートデータ内に「自分」が含まれない場合は、現在の設定が維持される。

### 一括エクスポート (Batch Export)
- **機能**: アプリ内のデータをJSON形式で書き出す。
- **操作**: 
  1. 「一括エクスポート」をタップすると、データの種類（スケジュールデータ / プロフィールデータ）を選択するダイアログが表示される。
  2. 種類を選択すると、ファイル保存画面が開く。
  3. 保存先とファイル名を決定し、保存を実行。
- **ファイル名初期値**: 
  - スケジュール: `ws_fighters_schedule.json`
  - プロフィール: `ws_fighters_profile.json`

### 全てのデータを削除 (Clear All Data)
- **機能**: データベース内の全てのデータを一括で削除する。
- **対象**: 全ての大会（ゲーム）、戦績、ファイター、デッキ。
- **操作**:
  1. 「全てのデータを削除」をタップすると、確認ダイアログが表示される。
  2. 削除を承認すると実行され、完了後に通知が表示される。

## 2. アプリについて (About)

### ヘルプ
- **機能**: アプリの基本的な使い方をチュートリアル形式で表示する。
- **構成**: `ViewPager2` を使用したスライド形式（全7ページ）。
- **内容**:
  1. 大会・戦績の追加
  2. 編集と削除（長押し操作）
  3. SNS共有
  4. プロフィールとデッキ
  5. 自分設定
  6. アナリティクス
  7. データ移行（インポート/エクスポート）
- **操作**: 画面下部の「前へ」「次へ」ボタンでページを移動。最終ページでは「閉じる」ボタンが表示される。

### このアプリについて
- **機能**: アプリの基本情報（バージョン、開発者）を表示する。
- **操作**: 
  1. 「このアプリについて」をタップすると、ダイアログが表示される。
  2. ダイアログにはバージョン番号と作成者名（ふぁいる）が表示される。

## 3. 開発者オプション (Developer Options)

### デバッグモード (Debug Mode)
- **機能**: ONにすると、インポート時にファイル選択の代わりに、アセット内のサンプルデータを自動で読み込む。
- **対象ファイル**:
  - スケジュール: `sample_schedule.json`
  - プロフィール: `sample_profile.json`
- **用途**: 開発時の迅速なデータ投入と動作確認を目的とする。
