# 不具合管理票: ゲーム形式変更によるDB不整合

## 概要 (Title)

チーム戦として作成したゲームにスコアを登録した後、そのゲームの形式をシングルスに変更すると、チーム戦のスコア情報が残ったままになり、データベースに不整合なデータが生まれてしまう。

## 再現手順 (Steps to Reproduce)

1.  対戦形式を「チームス」として、新しいゲームを作成する。
2.  作成したゲームに、対戦結果（スコア）を登録する。この時、「チームの戦績」も入力する。
3.  作成したゲームの編集画面を開く。
4.  対戦形式を「チームス」から「シングルス」に変更して保存する。

## 本来あるべき挙動 (Expected Behavior)

- ゲーム形式の変更によって、データの不整合が発生しないこと。

## 実際の挙動 (Actual Behavior)

- ゲームの形式は「シングルス」に変更されるが、それに紐づく`score`レコードには、本来シングルスではありえない`team_win_lose`のデータが残ってしまう。

## 提案される解決策 (Proposed Solution)

- **一度作成したゲームの対戦形式（`game_style`）は、編集できないようにする。**
- `RecordGameDialogFragment`（大会・イベントの編集ダイアログ）を開いた際に、対戦形式を選択するラジオボタンを無効化（disable）し、ユーザーが変更できないようにUIを修正する。
